js 개발 방식
- 성능 차이 없고, 단지 디자인 패턴의 차이
큰 구조로 2개
1. 객체형 프로그래밍 : 전통 강호 (class가 나오는 단어: 무시)
2. 함수형 프로그래밍 : 신흥 강자 

함수
매개변수를 집어넣었을 때 값이 나옴.
-> 특정 기능을 묶어놓음

함수의 역할
-> 하나의 뭉치로 쓰여진 코드를 논리적으로 나누어 줌
-> 한 함수에서 버그가 발생시 그 부분만 동작되지 않고, 나머지 코드는 정상적으로 구동 (js는 오류시 알아서 고쳐주지 않음)
-> 여러번 반복되는 코드를 하나로 줄여줘 유지보수가 간편
-> 코드의 가독성이 높아짐
